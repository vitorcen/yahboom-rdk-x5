组件关系

这台板子的相机不是普通的 /dev/video0 V4L2 摄像头，而是 Horizon VIN、ISP、VSE 与 ROS2 组成的专用通路。

01IMX219CSI0 上的 2-lane RAW10 sensor，控制地址位于 i2c-6:0x10。
02cam-service提供 /dev/isc 后台中间件。它必须保持运行，但本身不提供网页。
03mipi_cam配置 VIN、ISP、VSE，并向 ROS2 发布 /image_raw。
04hobot_codec将 NV12 转为 JPEG，发布 /image_jpeg。
05websocket + nginx在 8000 端口提供 TogetheROS Web 预览。
两个程序不能同时占用 CSI0：亚博桌面自启的 app_SunriseRobot.py 也会通过 libsrcampy 打开相机。运行 TogetheROS 预览前，需要先结束该程序。

怎么修复的

  1. 重新扫描硬件，确认 IMX219 已从错误插口移到正确的 CSI0 / i2c-6，地址为 0x10。
  2. 发现亚博桌面应用 app_SunriseRobot.py 会自动打开 CSI0，与 tros 的 mipi_cam 互斥。
  3. 修正原脚本中的错误假设：cam-service 不是相机占用者，而是 ISP/VSE 必需的后台服务，不能停止。
  4. 在 cam-service active、亚博 APP 已停止的状态下，用官方 API 成功抓取真实帧。
  5. 保留 cam-service，重新启动 mipi_cam_websocket.launch.py，最终得到约 30 FPS 和可访问的 Web 页面。
旧操作问题修正后
systemctl stop cam-serviceISP/VSE 配置失败，常见 ret(-10)systemctl start cam-service 并保持 active
保留亚博 APPCSI0 被占用，mipi_cam 无法独占采集启动 tros 前结束 app_SunriseRobot.py
只检查 8000 端口网页能打开不代表有图像同时检查 /image_raw 与 /image_jpeg 帧率

手工启动

项目提供统一模式切换脚本，并同步到板子的 /home/sunrise/work/scripts/。

ssh root@192.168.3.187

# 切到 TogetheROS 浏览器预览
sudo /home/sunrise/work/scripts/camera_mode.sh tros

# 切回亚博手机 APP（IP、控制端口、视频端口）
sudo /home/sunrise/work/scripts/camera_mode.sh yahboom

# TogetheROS 负责视频，亚博 APP 只负责遥控
sudo /home/sunrise/work/scripts/camera_mode.sh hybrid

# 查看当前模式和监听端口
sudo /home/sunrise/work/scripts/camera_mode.sh status

便捷入口：camera_preview.sh 等价于 camera_mode.sh tros；camera_yahboom.sh 等价于 camera_mode.sh yahboom。

脚本执行以下操作：

  1. 在 i2c-6 和 i2c-4 检查 sensor。
  2. 确保 cam-service 运行。
  3. 停止另一个模式的进程，等待 CSI0、ISP 和 VSE 设备节点释放；切回亚博时重启 cam-service，清除 tros 留在中间件中的 CSI 会话。
  4. 后台启动 TogetheROS 相机 Web launch。
  5. 通过 ros2 topic hz /image_raw 验证是否真实出图。
当前预览地址：http://192.168.3.187:8000/TogetheROS/

推荐：混合模式

camera_mode.sh hybrid 不修改亚博原厂代码，而是通过 yahboom_control_only.py 启动原控制程序并替换其 MIPI Camera 类。最终分工如下：

功能程序与入口说明
手机遥控亚博 APP，TCP 6000底盘、串口和遥控协议保持不变
RDK 视频TogetheROS，:8000/TogetheROS/CSI0、ROS 图像话题和 Web 预览
亚博 Video Port6500为兼容 APP 仍监听，但混合模式故意不向它提供 CSI 视频
协议不能混填：亚博 Video Port 使用 /video_feed MJPEG，TogetheROS 8000 使用自己的 WebSocket 页面，所以手机 APP 的 Video Port 不能直接填写 8000。

开机能否自启

当前配置：不能自动恢复 TogetheROS 预览。当前 cam-service 已设置 enabled，但 ROS launch 是手工后台启动。重启后，亚博 XFCE autostart 会再次启动 app_SunriseRobot.py 并占用 CSI0，8000 端口预览不会自动出现。

方案 A：保留当前行为

适合仍需使用亚博手机 APP 控车的场景。每次重启后按需运行 camera_mode.sh tros，需要手机 APP 时再运行 camera_mode.sh yahboom，无需重启。

方案 B：让 TogetheROS 预览开机自启

该方案会禁用亚博桌面 APP 的自启，因为两者不能同时独占 CSI0。先禁用亚博 APP：

sed -i 's/^Hidden=.*/Hidden=true/' \
  '/home/sunrise/.config/autostart/Start APP Program.desktop'

创建 /etc/systemd/system/mipi-camera-preview.service：

[Unit]
Description=RDK X5 MIPI camera TogetheROS preview
Requires=cam-service.service
After=cam-service.service network-online.target
Wants=network-online.target

[Service]
Type=simple
Environment=HOME=/root
ExecStartPre=-/usr/bin/pkill -TERM -f [a]pp_SunriseRobot.py
ExecStart=/bin/bash -lc 'source /opt/tros/humble/setup.bash && exec ros2 launch mipi_cam mipi_cam_websocket.launch.py'
Restart=on-failure
RestartSec=3

[Install]
WantedBy=graphical.target

加载并启用：

systemctl daemon-reload
systemctl enable --now cam-service
systemctl enable --now mipi-camera-preview.service
systemctl --no-pager status mipi-camera-preview.service
切换代价：启用方案 B 后，亚博手机 APP 的相机/遥控后台不会随桌面启动。要恢复原厂 APP，请按“恢复原厂 APP”一节操作。

恢复原厂亚博 APP

如果仅使用手工预览，直接重启即可，因为当前亚博 autostart 已恢复为 Hidden=false。

如果配置过 systemd 自启方案，执行：

systemctl disable --now mipi-camera-preview.service
sed -i 's/^Hidden=.*/Hidden=false/' \
  '/home/sunrise/.config/autostart/Start APP Program.desktop'
systemctl enable cam-service
reboot

重启后由亚博桌面程序重新占用 CSI0，TogetheROS 预览随之停止。
