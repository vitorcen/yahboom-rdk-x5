USB 全向麦音箱 · 全模态语音通道方案USB omnidirectional speakerphone as the broad audio I/O channel for omni-modal LLM

TL;DR：车上现有的 CI1302 语音模块是封闭的"关键词对讲机"——麦克风音频不出芯片、喇叭只能放固件里烧死的 TTS 词条，没有任何任意音频进出的通路（mp3 物理放不了）。全模态大模型需要的是原始音频流双工通道：加一个 USB 全向麦音箱一体（UAC 免驱 speakerphone），arecord 拿麦克风流、aplay/mpg123 播任意音频，即插即用。CI1302 降级保留为离线唤醒词 + 应急命令通道，分工合理。
EN: The CI1302 module is a closed keyword-matching device with no raw-audio path. Add a driverless USB Audio Class speakerphone for full-duplex raw audio; keep CI1302 as an offline wake-word / fallback command channel.

1. 现状盘点Current audio landscape on the robot

通道输入 (mic→host)输出 (host→speaker)判定
CI1302 语音模块（启英泰伦，串口/IIC 协议）✗ 音频只进芯片内部做离线识别，主机只收 5 字节协议帧 AA FF <type> <id> FB✗ 只能触发固件内预烧 TTS 词条（≤300 条，改一句话=云平台重做固件+烧录）不可用作宽泛 I/O
板载 ES8326 codec（RDK X5 耳机座，card 0 duplexaudio）△ 有 ADC，但没接麦克风△ 有 DAC，但没接喇叭；接有源小音箱即通备用输出
USB 全向麦音箱一体（本方案，待购）✓ 48kHz 原始流，全向拾音+降噪✓ 任意音频/mp3/TTS 流主通道

实测佐证（2026-07-18 板上）：CI1302 模块当前 USB 线只供电未枚举（dmesg 无新设备）；两个 USB 串口分别被雷达（ttyACM0）和底盘驱动（ttyUSB0）占用；40-pin i2c-5 上的 0x0d 是底盘罗盘 QMC5883L 而非语音模块。要用 CI1302 协议通道需换带数据的 Type-C 线（滑动开关拨右=STC8 串口）或按教程杜邦线接 IIC。

2. 选型要点What to buy — selection criteria

指标要求为什么
协议USB Audio Class (UAC1/2) 免驱Linux/aarch64 零驱动，aplay -l 直接可见；避开任何"需装驱动/APP"的型号
回声消除硬件 AEC 必选机器人喇叭和麦在同一机身上，播 TTS 时自听会污染 ASR；硬件 AEC 免去板上软件 AEC 的 CPU 开销
拾音全向 (360°)，标称 ≥2m 拾音距离，带降噪/AGC小车会动，人不会贴着说话；雷达/电机有底噪
采样capture ≥16kHz（48kHz 更佳）、playback 48kHz 立体声16kHz 是主流 ASR/omni 模型输入下限
供电USB bus-powered ≤500mA，或自带电池挂在已有的独立供电 hub 上，不与 Astra 抢电
形态参考会议全向麦（EMEET / Anker PowerConf / Jabra Speak 同类，百元级国产亦可）一体机集成 mic 阵列+AEC+功放喇叭，一根线解决双工

4. 集成步骤Integration steps once hardware arrives

# 1) 插上后确认枚举为 USB 声卡（假设 card 1）
aplay -l ; arecord -l ; ls /dev/snd/by-id/

# 2) 固定为默认设备（按 by-id 而非卡号，防重枚举漂移）
cat > /etc/asound.conf <<'EOF'
defaults.pcm.card 1
defaults.ctl.card 1
EOF

# 3) 输出验收：放一个 mp3（用户最初的验收标准）
apt install -y mpg123 && mpg123 test.mp3

# 4) 输入验收：录 5 秒回放，人耳检查底噪/增益
arecord -f S16_LE -r 16000 -c 1 -d 5 /tmp/t.wav && aplay /tmp/t.wav

# 5) AEC 验收：aplay 播音乐的同时 arecord，录音里音乐残留应明显衰减

# 6) audio_io ROS2 节点 + systemd（仿 astra-cam.service 模板：
#    Restart=on-failure + 无声看门狗自杀重启，恢复热插拔）

5. 带宽 / 供电 / 风险Bandwidth, power, risks

项评估
USB2 带宽48kHz 16bit 立体声双工 ≈ 3 Mbps，相对 Astra 深度流（~18MB/s 上限的总线）可忽略；音频是 isochronous 有保留带宽，不会被深度流饿死
供电挂独立供电 hub（Astra 同款做法）；bus-powered 全向麦典型 200–400mA，别直插板载口与相机抢 500mA 预算
自听回声硬件 AEC 是选型硬指标；若买到无 AEC 型号，需在板上跑 speexdsp/webrtc-aec，8 核 A55 可承受但增加延迟
重枚举与雷达同款风险：audio_io 节点做无声看门狗（N 秒无帧 exit → systemd 重启），asound 按 by-id 绑定
行驶底噪电机/麦轮噪声进 mic；先静止对话验收，行驶中对话效果实测后再调（降噪档位/VAD 阈值）
隐私常开麦上云需明确开关（GUI 总闸），默认唤醒后才推流
