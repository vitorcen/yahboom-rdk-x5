RDK X5 控制台 GUI 架构 Tauri GUI Architecture

gui/ 是一个 Tauri 2 桌面应用：前端零依赖（gui/ui/： index.html 纯结构 + style.css + js/ 按职责拆模块——ros.js 连接与话题分发注册表、 viewer.js 渲染与导航、camera.js 相机窗、teleop.js 键盘遥控、health.js 仪表条、 system.js 系统页板卡框图、logs.js 日志页、main.js 装配；编译时嵌入二进制），后端是一层很薄的 Rust （10 个 async command：9 个 ssh 包装 + 1 个 ping 探活）。它与小车之间只有两条通信通道： rosbridge WebSocket（ws://192.168.3.187:9090）承载所有实时数据与控制指令， SSH（192.168.3.187:22，免密公钥）承载日志与服务状态查询。没有第三条。
Two channels only: rosbridge WebSocket (:9090) for all realtime data & control; SSH (:22) for logs & service status.

顶栏「主机在线」灯本机 ping -c1 -W1（ICMP，每 3 s）独立于 rosbridge——sshd/ROS 起来之前就能看到板子活没活。发电源命令后进入状态机：黄"重启中"直到观察到掉线→回来跳变才回绿（systemd 停 Nav2 栈约 1 min 内 ping 仍通，裸 ping 会误示"在线"）；关机则掉线后定格"已关机"探测 ←
顶栏「连接」按钮重建到 ws://…:9090 的 WebSocket；断线每 3 s 自动重连—
⚠️ 地图模式下画布上任何"按下-拖动-松开"都会真实发布 /goal_pose，车会动。 误触后立即点 ■ 终止。Nav2 的终止走的是 action cancel 服务，不是杀进程； 零速帧则兜底刹住底盘（经 cmd_vel_mux 仲裁）。

3. 总体框图 Data-flow diagram

蓝实线=数据前向流（DDS→WS），绿虚线=控制反向流（目标/速度），黄点线=SSH 旁路（日志）。 高亮框是本项目自研的两处关键件：cmd_vel_mux（安全仲裁）与 rosbridge/前端这条可视化主干。

4. 为什么这样设计 Design rationale

决策原因
可视化走 rosbridge，不跑板端 rviz2板端 rviz2 在 X 上必段错误（Ogre/GL，软件渲染也崩）；WebSocket+JSON 一条通道就够，前端零依赖可同时被浏览器和 Tauri 复用
日志走 SSH 而不走 rosbridgejournalctl/dmesg/systemctl 不是 DDS 数据；/rosout 只覆盖 ROS 节点，服务崩了（比如雷达假活）恰恰要看 journald。SSH 命令做了服务名白名单，不拼接任意输入
Rust 命令全部 async + spawn_blockingTauri 同步 command 跑在主线程，一次 ssh 1–4 s 会冻住整个窗口（踩过）
相机传 JPEG 帧不传 H.264板端 :8000 本来就是 MJPEG-over-WebSocket；/image_jpeg 是硬编码现成话题，46 KB/帧×15 fps ≈ 7.4 Mbps 家用 WiFi 无压力，前端一个 <img> 就能解码
终止 = cancel 服务 + 零速连发只 cancel 不刹车：DWB 最后一帧速度还在跑；只发零速不 cancel：bt_navigator 会重新规划。两个都要
已知隐患（Task #8）：雷达 USB 开机后偶发重枚举（ttyACM0→ttyACM1），oradar_scan 握着旧 fd「假活」——服务 active 但 /scan 无数据，AMCL 因此拿不到 map TF。日志页 journald 视图 就是为这类「/rosout 里看不见的死法」准备的。临时恢复：systemctl restart ms200-lidar。

5. 构建与运行 Build & run

依赖：rustup（stable）+ libwebkit2gtk-4.1-dev。 一键构建并运行：./gui/run.sh（发布版加 --release，已配 strip+LTO；前端改动会触发重编，build.rs 里有 rerun-if-changed=../ui）。前端也可脱离 Tauri 用浏览器打开——ES 模块受 file:// 限制，需 cd gui/ui && python3 -m http.server 8899 后访问 http://127.0.0.1:8899/—— 仪表盘全功能可用，仅日志页的 journald 部分需要 Tauri 后端。
