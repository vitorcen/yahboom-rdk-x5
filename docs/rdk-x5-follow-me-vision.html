Follow-me 视觉跟随Vision-only person following

RDK X5 Robot · 单目 IMX219(上仰视角) · v2 2026-07-12(经外部评审收敛为 MVP,已实现并台架验证)
TL;DR — 不训练任何模型。板上 TogetheROS 自带 BPU 加速的 mono2d_body_detection(人体/头/脸/手 检测框 + 跟踪ID)和 hand_gesture_detection(手势分类),已实机跑通(感知链约 22–30 FPS,单帧 BPU 推理 ~10ms)。 自研部分两个文件:follow/follow_me.py(约 200 行,OK 锁定 / Palm·SELECT·丢失·感知超时 停止, 相对尺度控制)+ 升级版 nav_config/cmd_vel_mux.py(三优先级仲裁 + 独立于跟随进程的雷达前向钳位)。 丢目标即停车解锁,重新 OK 才恢复;不做米制测距,不做颜色重识别。 Zero training; on-board BPU perception; latched stops; relative-scale control; independent lidar guard.
v2 评审修订记录(外部评审 gpt-5.6-sol,全部采纳)——相对 v1 的关键改动:
  • 砍掉颜色重识别与 SEARCH 态:跟错人比丢人危险。丢跟踪ID → 立即刹停并解锁,重新 OK 才恢复。
  • 砍掉米制测距与三点标定:锁定瞬间的框尺寸即基准,控制误差 e = log(ref/now),只需相对远近。
  • 转向改用 body 框中心(侧脸时脸框先消失);脸/头框只做尺度,且触及画面上沿视为无效(被裁剪,d=k/h 会反向失效)。
  • 纠正安全措辞:手柄 SELECT 是软件仲裁(mux 高优先级 + joy_ctrl 使能),不是硬通道;贴身实测应备物理断电手段。
  • 雷达钳位从"可选"改为必选,且实现在 mux 进程内、独立于跟随节点:scan 过期(>0.4s)禁一切自主运动;前扇区 <0.35m 禁前进;永远禁倒车。
  • 感知 watchdog:跟随中 >0.3s 无感知消息 → 锁存停车。刹停旁路加速度限幅,立即生效。
  • 横向偏差大只转不走;前进加速度限幅 0.3 m/s²;死区消抖。
  • 声光异步:蜂鸣由定时器排程,永不阻塞控制;放弃 RGB——驱动的 RGBLight 回调裸写 I2C,有崩溃前科。
  • 多人同时 OK → 拒绝锁定,绝不猜。

三、手势协议与声反馈Gestures & feedback

手势 / 事件动作蜂鸣
OK 👌(值 11,连续 3 帧,且全场仅一人做)锁定该人,开始跟随;多人同时 OK 一律拒绝两短鸣(嘀·嘀)
手掌 ✋(值 5,连续 2 帧,任何人)停止跟随(安全优先,不验身份),锁存一长鸣
手柄 SELECT(buttons[6])停止跟随,锁存一长鸣
目标丢失 >0.6s / 感知断流 >0.3s刹停并解锁(丢失瞬间即发零速,0.6s 只是身份宽限)三短鸣

所有停止都是锁存的:只有新的 OK 手势能重新进入跟随。蜂鸣序列由 50ms 定时器驱动,失败不影响停车。RGB 不用(驱动 I2C 崩溃前科)。

四、认人与测距:全部相对化Re-ID & ranging, all relative

  • 认人 = 跟踪ID,只有跟踪ID。检测端 MOT 的 track_id 在,就跟;不在,就停。没有颜色猜人、没有找回漫游——v1 的衣服直方图方案经评审否决:最危险的失败不是丢人,是跟错人后继续走;且 PerceptionTargets 不含像素,做直方图还得另订图像流+时间戳对齐,复杂度远超收益。
  • 测距 = 锁定瞬间的框尺寸做基准,误差 e = log(ref/now):框比基准小 → 人远了 → 前进;不需要米、不需要标定、天然适配不同身高。尺度观测按 脸高 → 头高 → body 宽 降级(各存各的基准);脸/头框贴画面上沿(y_offset≤2px)即视为被裁剪、无效——上仰视角下"近←→框大"在裁剪后会反向,宁可判无效停车。
  • 转向 = body 框中心横向偏差。body 框在侧脸、背对时依然稳定;脸框绝不参与转向。

五、控制律与状态机Control & state machine

ex = body中心x/960 − 0.5                  # 横向偏差
wz = −1.6·ex   死区 |ex|≤0.04,钳位 ±1.0 rad/s
e  = log(ref/now)                          # 尺度误差(远为正)
v  = 0.5·e     仅当 e > 0.10(死区);随 |ex| 线性衰减,|ex|≥0.18 时 v=0(只转不走)
v  加速限幅 0.3 m/s²(减速/刹停不限,立即生效),钳位 [0, 0.25] m/s —— 永不倒车
发布 /cmd_vel_follow,随感知帧率(~22Hz)

六、安全设计(实测过的和明说的盲区)Safety, verified & known gaps

  • mux 内雷达钳位,独立于跟随进程(已实测):scan 过期 >0.4s → follow 输入整体清零;前扇区(±30°)最近点 <0.35m → 禁前进、允许原地转;任何情况禁倒车。实测抓到一个经典坑:前扇区全为无效回波(0.0)时曾被当"畅通"放行——MS200 在 ≤0.1m 盲区和吸光面正好返回无效值,已改为"无有效数据=有障碍"。
  • 手柄 SELECT 是软件仲裁不是硬通道:joy 高优先级压制 + follow 节点收到 SELECT 锁存停止。贴身测试请准备物理断电(电源开关就在车尾)。
  • Palm 任何人可停,不验身份。
  • 已知盲区(2D 雷达一个平面):低矮物、悬空物、桌角、台阶、下行楼梯全都看不见;测试场地先清场。倒车已禁,侧向碰撞靠"只转不走"的原地旋转半径兜底,非零风险。
