RDK X5 Robot · IMX219 单目(上仰) + ORADAR MS200 · v3 2026-07-12 · 实测可用
TL;DR — 零训练。板上 BPU 感知(人体/手势检测 ~30FPS)出身份,雷达腿部聚类出几何, 两通道常开并行(不是模式切换):方位优先用相机(带身份),距离优先用雷达(比框宽准一个量级), 任一通道黑掉另一路无缝顶上。麦轮底盘按速度矢量直指主人(vx=v·cosθ, vy=v·sinθ), 车头对齐只是并行姿态修正——运动方向永远不会错。 桌凳腿靠世界系运动判别(odom 抵消自身运动)排除:不动的不是人。 👌OK 锁定 / ✋Palm 停止,GUI 滑块开关经 systemd 持久化,小车重启自动恢复。 Dual always-on channels; velocity vector aims at the owner; world-frame motion test rejects furniture.

三、融合跟随算法The fusion algorithm

3.1 双通道观测(常开并行)

相机通道(30Hz)雷达通道(10Hz)
提供身份(track_id)、手势、body 方位角、肩宽估距 d≈360px·m/宽px(粗)腿部聚类精确 (方位, 距离);全 360°(±π 接缝合并,正后方不再是盲区)
有效期0.4s1.2s(单腿视角会闪烁)
失效时雷达腿轨迹顶上(近距/急转/背对全覆盖)相机肩宽估距兜底

雷达门限的锚定是融合的关键:相机新鲜时,聚类搜索门限以相机方位+肩宽估距为中心(±0.5rad, ±0.8m)——雷达永远知道"腿该在哪";相机黑掉后,门限改用腿轨迹自身+速度外推(±0.4rad, ±0.4m,速度是低通滤波的方位/距离变化率)——人在走,门就跟着人走。

3.2 认腿:尺寸只是入场券,运动才是身份

  • 聚类:相邻点角差 <0.06rad 且距差 <0.10m 归组;物理宽 4–40cm(单腿到双腿并拢)为候选。桌凳腿同样通过——尺寸无法区分。
  • 运动判别(世界系):所有候选聚类经 /odom 换算到世界坐标,滚动保留 1.2s 快照,机器人自身的移动/旋转被抵消。两条规则:
    1. 咬合关(相机黑时新抓目标):候选位置 1.2s 前已有东西占着 → 家具,拒绝。odom 不新鲜时一律不盲抓。
    2. 持有关:跟踪中的"腿"世界系 3s 纹丝不动(<0.12m)且相机黑 → 判家具,放弃站住(日志"疑似家具,放弃")。相机能看到 body 时不做此淘汰——站着的人是合法目标。
  • ID 收养:转身使外观突变,跟踪器可能换发新 ID。旧 ID 消失时,若有 body 与旧位置重叠(横移 <25% 画宽、宽度比 0.5–2)或与腿轨迹方位吻合(±0.3rad),锁直接转移,不判丢失。

3.3 控制律:矢量速度 + PD 转向,10–40Hz

方位 θ:相机新鲜用相机(身份保证),否则用腿轨迹
距离 d:腿轨迹新鲜用雷达实测(基准 ref_dist 在锁定后第一次雷达命中时记下),
        否则退回相机框尺寸比 e=log(ref/now)

速度大小  v = KV·(d − ref_dist)          死区 0.07m,加速限幅 0.8 m/s²(刹车不限)
速度上限  cap = 0.5 + 0.3·min(|θ|/0.6rad, 1)   直行 0.5,角度越大越快,封顶 0.8
速度方向  vx = v·cosθ, vy = v·sinθ       ← 麦轮全向:方向瞬时指向主人,与车头解耦
转向     wz = 4.0·θ + 0.6·θ̇             PD:变化率前馈,主人一横移车头就起转
控制节拍  雷达帧(10Hz)+ 相机帧(30Hz)都踩控制 ≈ 40Hz,方位延迟 ≤33ms

"前冲方向错、越追越远"的旧病根:速度只能沿车头发,车头没转到位方向就是错的。矢量化后车头对齐(wz)只是并行的姿态整理,不再是速度的前提。

事件判定(滑窗投票,抗分类闪烁)动作 / 蜂鸣

四、安全Safety

  • mux 内钳位,独立进程,方向感知:跟随速度是矢量,钳位查运动方向 ±30° 扇区的雷达余量,按 允许速度=(余量−0.35m)/0.3 比例限速(无开关震荡);扇区无有效回波=堵死;scan 过期 >0.4s 全零;vx<0 永远禁止。斜滑不是盲区。
  • 手柄最高优先级:joy 压制 + follow 节点见 SELECT 锁存。注意这是软件仲裁非硬通道,贴身实测备好车尾电源开关。
  • 已知盲区:2D 雷达单平面——低矮物/悬空物/台阶不可见;运动判别无法区分"相机外站着的人"和柜子(3s 静止会松口,人挪步即重咬)。

systemd 部署踩过的坑:①脚本 trap 'kill 0' 会把自己也杀掉,systemd 判 signal 失败→无限重启,要只杀后台子进程;②service 环境无 HOME,ROS 日志目录展开失败节点秒崩,需 Environment=HOME=/root + ROS_LOG_DIR=/tmp/roslog。
