原理 / How it works

数据链路：MS200 雷达经串口（/dev/oradar → ttyACM0）进入板端 oradar_scan 驱动节点，以 10 Hz 发布 /scan；底盘 Mcnamu_driver+EKF 发布 /odom 与 TF； Cartographer 消费 /scan+/odom 增量建图，occupancy_grid 节点以 1 Hz 发布 /map；Nav2 导航时再发布 /plan。rosbridge_server 把这些 DDS 话题桥接成 WebSocket（:9090）+ JSON。本页面用手写的最小 rosbridge v2 客户端（原生 WebSocket，零外部依赖）订阅上述话题渲染到 Canvas；反向还会 advertise /goal_pose——在地图上拖一下即发布导航目标，Nav2 收到后驱动底盘。
Pipeline: serial → driver → cartographer/Nav2 → rosbridge (DDS→WebSocket/JSON) → hand-rolled JS client → Canvas; goals flow back via /goal_pose.

渲染细节：/map 是 nav_msgs/OccupancyGrid（row-major，-1 未知 / 0 空闲 / 100 占用）， 按 origin+resolution 画到离屏 canvas 再缩放贴图（世界 y 向上 → 屏幕 y 向下翻转）。机器人位姿由 /tf + /tf_static 在页面内组合（map→odom→base_footprint，2D 平面只取 yaw）。雷达点按 map→lidar_link 变换叠加。为什么不用板端 rviz2：它在本板 X 上稳定段错误（Ogre/GL）。 启动/停止见 README §8.2。
