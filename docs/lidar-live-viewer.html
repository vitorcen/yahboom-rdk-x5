原理 / How it works

数据链路：MS200 雷达经串口（/dev/oradar → ttyACM0）进入板端 oradar_scan 驱动节点，以 10 Hz 发布 ROS 2 话题 /scan（sensor_msgs/LaserScan，450 点/圈）。 rosbridge_server 在板上把 DDS 话题桥接成 WebSocket（:9090）+ JSON。本页面用 手写的最小 rosbridge v2 客户端（原生 WebSocket，零外部依赖、无 CDN）发送一条 {op:"subscribe", topic:"/scan"}，之后每帧 publish 消息到达即重绘 Canvas。
Pipeline: serial → ROS 2 driver → rosbridge (DDS→WebSocket/JSON) → hand-rolled JS client → Canvas.

为什么不用板端 rviz2：rviz2 在本板 X 桌面稳定段错误（Ogre/GL 栈，软件渲染也崩）。 浏览器方案帧率更高、跨平台、无需 VNC。启动/停止命令见仓库 README §8.2。
Why not rviz2 on the board: it consistently segfaults (Ogre/GL); this browser viewer is lighter and needs no VNC. See README §8.2 for start/stop commands.
