遛狗模式(2026-07-14,`nav_config/dog_walk.py`,已部署板上、**未实机验证**——电池 7.3V 欠压):
小车像小狗一样走走停停闻闻,朝空旷方向三角波变速小跑,遇桌脚凳脚一类孤立小障碍凑近"闻"几秒、
后退一小步、转身去下一处。**纯反应式,无地图/无 Nav2/无 odom** —— 就是单帧 MS200。
设计文档 `docs/rdk-x5-dog-walk-design.html`(内嵌拓扑/状态机 SVG)。

**流程**:先写设计 HTML → `codex exec -m gpt-5.6-sol` 评审 → Opus 子 agent 实现 → 我逐条把关。
codex 推翻 3 处初稿坏品味(都采纳):①"共用 LOW `/cmd_vel` 不改 mux"是假简洁 → 独立通道;
②"抓杆临时让权后自动续跑"是人机语义错 → **抓杆即退出 active**(订阅 `/cmd_vel_joy` 非零即自停);
③单帧腿判据太脆 → **物理宽度 [0.02,0.3]m + 最少点数 + 连续 3 帧一致**(不用 odom 验证"静止")。

**关键设计**:
- **mux 优先级:joy > dog(P1) > follow(P2) > nav(P3)**。遛狗排在跟随/Nav2 之上,是为"**停动作不停功能**":
  启遛狗即在总线层压住跟随的追人/Nav2 的目标运动,但**功能开关/服务原样在**,遛狗一停立刻续上。
  guard 随 follow 移到 `prio==2`;dog 无 guard(要后退,MID guard 拒 vx<0)。反向后到抢占用单次
  信号:开跟随/点导航目标发 `/dog_stop` 停狗;启狗顺手 cancel Nav2 目标。**别再要求先关跟随**。
- **无急刹门禁**:遛狗本就是雷达驱动(自带扇区避障 + scan>0.4s 陈旧即 fail-close 停),不依赖也不碰
  safety_stop 开关。(曾按 codex 加过"急刹关则拒启",用户否掉——雷达控制的东西不随便碰急刹。)
- **贴近 ~20cm 闻的几何**:停距 0.32m 是**雷达到物体**距离,雷达在车中段→**车头离物体 ~20cm**,
  紧贴 safety_stop 0.3m 地板,不用降急刹就能贴近。
- **R1=buttons[7] 切换,R2=buttons[9] 抓到待定**(此 pad 从不按惯例,见 [[rdk-x5-gamepad-teleop]])。
  GUI 🐕 按钮异步(点击即"启动中…"过渡,`/dog_active` latched 敲定并**跳秒显示已跑时长**);
  日志页订阅 `/joy` 打印按键下标=抓 R1/R2 的探针。10min 硬超时,启/停=两短滴/一长滴。

**活泼化教训**:"整体太慢不活泼"根因不是 0.5 上限,是**净空限速太保守**——旧版要 1.2m 净空才给满速,
杂乱房间净空常 0.6~1.0m 被压到 40~60% × gait 下限 → 实际才 0.15~0.25。改**满速点 0.8m**
(`clamp((clear−0.35)/0.45)`)+ gait 下限 0.55→0.72/周期 8s + 转向 WZ/TURN/K 提到 1.6/1.3/1.9。

**欠压再教训**(2026-07-14):测遛狗时出"纯手柄推杆松开不停、按第二次才停"的幻影 teleop bug,
排查代码全对、idle 全 0——最后查 `/voltage=7.3V`。正是 [[rdk-x5-nav2-plan]] 记的"欠压时软件链路
查什么都正常,唯一正确动作是充电"(驱动 I2C 崩→MCU 续发速度→respawn 恢复)。**先查电压再查代码**。
